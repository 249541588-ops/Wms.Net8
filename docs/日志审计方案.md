# .NET 8 操作日志/系统日志 — 主流方案汇总

## 背景

MES 系统需要记录系统日志和操作日志，用于排查问题、审计追踪和合规要求。

---

## 方案一：数据库表 + ActionFilter（推荐 ★★★）

### 原理
全局 `IAsyncActionFilter` 拦截所有 API 请求，写入 `SysOperationLog` 数据库表。

### 实现步骤

#### 1. 创建日志实体

```csharp
// Domain/Entities/SysOperationLog.cs
public class SysOperationLog : IEntity<long>, IAuditable
{
    public long Id { get; set; }
    public string? Module { get; set; }        // 模块（Controller名）
    public string? Action { get; set; }        // 操作（Action名）
    public string? Method { get; set; }        // HTTP方法（GET/POST/PUT/DELETE）
    public string? Url { get; set; }           // 请求地址
    public string? RequestParams { get; set; } // 请求参数（JSON）
    public string? ResponseResult { get; set; }// 响应结果（可选）
    public int StatusCode { get; set; }        // HTTP状态码
    public long Duration { get; set; }         // 耗时（ms）
    public string? IpAddress { get; set; }     // 客户端IP
    public string? UserAgent { get; set; }     // 浏览器信息
    public string? UserId { get; set; }        // 操作人ID
    public string? UserName { get; set; }      // 操作人姓名
    public DateTime CreatedTime { get; set; }
    public DateTime ModifiedTime { get; set; }
    public string? CreatedBy { get; set; }
    public string? ModifiedBy { get; set; }
}
```

#### 2. EF Core 配置

```csharp
// Infrastructure/Persistence/Configurations/SysOperationLogConfiguration.cs
public class SysOperationLogConfiguration : IEntityTypeConfiguration<SysOperationLog>
{
    public void Configure(EntityTypeBuilder<SysOperationLog> builder)
    {
        builder.ToTable("SysOperationLog");
        builder.HasKey(x => x.Id);
        builder.HasIndex(x => x.CreatedTime);
        builder.HasIndex(x => x.UserId);
        builder.Property(x => x.RequestParams).HasMaxLength(4000);
        builder.Property(x => x.ResponseResult).HasMaxLength(4000);
    }
}
```

#### 3. 全局 ActionFilter

```csharp
// WebApi/Filters/OperationLogFilter.cs
public class OperationLogFilter : IAsyncActionFilter
{
    private readonly IServiceScopeFactory _scopeFactory;
    private readonly ILogger<OperationLogFilter> _logger;

    public OperationLogFilter(IServiceScopeFactory scopeFactory, ILogger<OperationLogFilter> logger)
    {
        _scopeFactory = scopeFactory;
        _logger = logger;
    }

    public async Task OnActionExecutionAsync(ActionExecutingContext context, ActionExecutionDelegate next)
    {
        var stopwatch = Stopwatch.StartNew();
        var result = await next();
        stopwatch.Stop();

        try
        {
            var httpContext = context.HttpContext;
            var request = httpContext.Request;
            var routeValues = context.ActionDescriptor.RouteValues;

            var log = new SysOperationLog
            {
                Module = routeValues["controller"],
                Action = routeValues["action"],
                Method = request.Method,
                Url = $"{request.Path}{request.QueryString}",
                StatusCode = result.HttpContext.Response.StatusCode,
                Duration = stopwatch.ElapsedMilliseconds,
                IpAddress = httpContext.Connection.RemoteIpAddress?.ToString(),
                UserAgent = request.Headers["User-Agent"].ToString(),
                UserId = httpContext.User?.FindFirst(ClaimTypes.NameIdentifier)?.Value,
                UserName = httpContext.User?.FindFirst(ClaimTypes.Name)?.Value,
                CreatedTime = DateTime.Now,
                ModifiedTime = DateTime.Now,
            };

            // 请求参数
            if (context.ActionArguments.Count > 0)
            {
                log.RequestParams = JsonSerializer.Serialize(context.ActionArguments);
            }

            // 异步写入，不阻塞响应
            _ = Task.Run(async () =>
            {
                using var scope = _scopeFactory.CreateScope();
                var db = scope.ServiceProvider.GetRequiredService<MesDbContext>();
                db.Set<SysOperationLog>().Add(log);
                await db.SaveChangesAsync();
            });
        }
        catch (Exception ex)
        {
            _logger.LogError(ex, "记录操作日志失败");
        }
    }
}
```

#### 4. 注册 Filter

```csharp
// Program.cs
builder.Services.AddScoped<OperationLogFilter>();

// 或者全局注册
builder.Services.AddControllers(options =>
{
    options.Filters.AddService<OperationLogFilter>();
});
```

#### 5. 数据库迁移

```sql
CREATE TABLE SysOperationLog (
    Id BIGINT IDENTITY(1,1) PRIMARY KEY,
    Module NVARCHAR(100) NULL,
    [Action] NVARCHAR(100) NULL,
    Method NVARCHAR(10) NULL,
    Url NVARCHAR(500) NULL,
    RequestParams NVARCHAR(4000) NULL,
    ResponseResult NVARCHAR(4000) NULL,
    StatusCode INT NOT NULL DEFAULT 0,
    Duration BIGINT NOT NULL DEFAULT 0,
    IpAddress NVARCHAR(50) NULL,
    UserAgent NVARCHAR(500) NULL,
    UserId NVARCHAR(64) NULL,
    UserName NVARCHAR(64) NULL,
    CreatedTime DATETIME NOT NULL,
    ModifiedTime DATETIME NOT NULL,
    CreatedBy NVARCHAR(64) NULL,
    ModifiedBy NVARCHAR(64) NULL
);
CREATE INDEX IX_SysOperationLog_CreatedTime ON SysOperationLog(CreatedTime);
CREATE INDEX IX_SysOperationLog_UserId ON SysOperationLog(UserId);
```

### 优点
- 简单直接，不依赖第三方
- 灵活可定制（可过滤特定接口、添加业务字段）
- 可在前端展示日志查询页面
- 异步写入，不影响接口性能

### 缺点
- 需自建表和查询页面
- 大量日志时数据库增长较快（需定期清理）

### 适用场景
- 中小型项目
- 需要操作审计（谁做了什么）
- 需要前端日志管理页面

## 方案二：Serilog + Seq/Elasticsearch（结构化日志平台）

### 原理
Serilog 作为日志框架，将结构化日志发送到 Seq 或 Elasticsearch，通过 Web UI 查询分析。

### 实现步骤

#### 1. 安装 NuGet

```bash
dotnet add package Serilog.AspNetCore
dotnet add package Serilog.Sinks.File
dotnet add package Serilog.Sinks.Async
# 可选：Seq
dotnet add package Serilog.Sinks.Seq
# 可选：Elasticsearch
dotnet add package Serilog.Sinks.Elasticsearch
```

#### 2. 配置 Program.cs

```csharp
// Program.cs
Log.Logger = new LoggerConfiguration()
    .ReadFrom.Configuration(builder.Configuration)
    .Enrich.FromLogContext()
    .Enrich.WithMachineName()
    .Enrich.WithThreadId()
    .CreateLogger();

builder.Host.UseSerilog();
```

#### 3. appsettings.json 配置

```json
{
  "Serilog": {
    "Using": ["Serilog.Sinks.File", "Serilog.Sinks.Async"],
    "MinimumLevel": {
      "Default": "Information",
      "Override": {
        "Microsoft.EntityFrameworkCore": "Warning",
        "Microsoft.AspNetCore": "Warning"
      }
    },
    "WriteTo": [
      {
        "Name": "Async",
        "Args": {
          "Configure": [
            {
              "Name": "File",
              "Args": {
                "path": "Logs/log-.txt",
                "rollingInterval": "Day",
                "retainedFileCountLimit": 30,
                "outputTemplate": "{Timestamp:yyyy-MM-dd HH:mm:ss.fff} [{Level:u3}] [{SourceContext}] {Message:lj}{NewLine}{Exception}"
              }
            }
          ]
        }
      }
    ]
  }
}
```

#### 4. 使用示例

```csharp
// 在 Service 中直接使用 ILogger
public class TProductService
{
    private readonly ILogger<TProductService> _logger;

    public Result CreateProduct(...)
    {
        _logger.LogInformation("创建产品：{ProductCode}, 操作人：{User}", productCode, user);
        // ...
        _logger.LogError(ex, "创建产品失败：{ProductCode}", productCode);
    }
}
```

### 优点
- 结构化日志，支持 JSON 格式
- 多 Sink 输出（文件、Seq、ES、Slack）
- 按日期/大小自动滚动
- 与 Seq/Kibana 集成可实现强大的日志分析

### 缺点
- 需部署 Seq 或 ES 服务（增加运维成本）
- 日志查询需依赖第三方 UI

### 适用场景
- 中大型项目
- 需要日志分析和可视化
- 微服务架构

## 方案三：Audit.NET（专业审计库）

### 原理
通过 EF Core 拦截器，自动记录所有数据库变更的详细信息（字段级）。

### 实现步骤

#### 1. 安装 NuGet

```bash
dotnet add package Audit.NET
dotnet add package Audit.NET.EntityFrameworkCore
```

#### 2. 配置

```csharp
// Program.cs
Audit.Core.Configuration.Setup()
    .UseSqlServer(_ => _
        .ConnectionString(builder.Configuration.GetConnectionString("Default"))
        .TableName("AuditLog")
        .IdColumnName("Id")
        .JsonColumnName("Data")
        .LastUpdatedColumnName("LastUpdatedOn"))
    .WithCreationPolicy(EventCreationPolicy.InsertOnEnd);

// DbContext 中启用
builder.Services.AddDbContext<MesDbContext>(options =>
{
    options.UseSqlServer(connectionString)
           .AddInterceptors(new AuditSaveChangesInterceptor());
});
```

#### 3. 在实体上标记审计

```csharp
[AuditInclude]  // 标记需要审计的实体
public class TProduct : IEntity<int>, IAuditable
{
    // ...
}
```

### 优点
- **字段级变更记录**：记录修改前/修改后每个字段的值
- 零代码侵入（标记特性即可）
- 支持多种存储目标

### 缺点
- 高并发场景有性能开销
- 配置稍复杂
- 日志数据量大

### 适用场景
- 需要详细审计追踪（金融、医疗等合规要求）
- 需要记录字段级变更

## 方案四：MediatR Pipeline Behavior（DDD/CQRS 架构）

### 原理
通过 MediatR 的 `IPipelineBehavior<TRequest,TResponse>` 拦截所有 Command 请求。

### 实现步骤

```csharp
// Pipeline/LoggingBehavior.cs
public class LoggingBehavior<TRequest, TResponse> : IPipelineBehavior<TRequest, TResponse>
    where TRequest : IRequest<TResponse>
{
    private readonly ILogger<LoggingBehavior<TRequest, TResponse>> _logger;

    public LoggingBehavior(ILogger<LoggingBehavior<TRequest, TResponse>> logger)
    {
        _logger = logger;
    }

    public async Task<TResponse> Handle(TRequest request, RequestHandlerDelegate<TResponse> next, CancellationToken cancellationToken)
    {
        var requestName = typeof(TRequest).Name;
        _logger.LogInformation("处理请求：{RequestName}，参数：{@Request}", requestName, request);

        var response = await next();

        _logger.LogInformation("请求完成：{RequestName}", requestName);
        return response;
    }
}
```

### 优点
- 与 CQRS 架构天然契合
- 只记录有意义的业务操作
- 可结合 FluentValidation 做请求验证

### 缺点
- 需要项目已使用 MediatR
- 只记录 Command，不记录 Query

### 适用场景
- DDD/CQRS 架构项目

## 方案六：NLog（高性能结构化日志）

### 原理
与 Serilog 齐名的 .NET 日志框架（GitHub 6.3K Stars），支持结构化日志，异步写入性能更优。

### 与 Serilog 对比

| 特性 | NLog | Serilog |
|------|------|---------|
| 结构化日志 | ✅ | ✅ |
| 性能 | 更优（异步写入更快） | 优 |
| 配置方式 | XML/代码/配置文件 | 代码优先 |
| Sink/Target 数量 | 100+ | 300+ |
| 学习曲线 | 低 | 中 |

### 实现步骤

#### 1. 安装 NuGet

```bash
dotnet add package NLog.Web.AspNetCore
```

#### 2. 配置 Program.cs

```csharp
// Program.cs
builder.Logging.ClearProviders();
builder.Logging.SetMinimumLevel(LogLevel.Trace);
builder.Host.UseNLog();
```

#### 3. nlog.config 配置文件

```xml
<?xml version="1.0" encoding="utf-8" ?>
<nlog xmlns="http://www.nlog-project.org/schemas/NLog.xsd"
      xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
      autoReload="true"
      throwExceptions="false">

  <targets>
    <!-- 异步写入文件（按日期滚动） -->
    <target xsi:type="AsyncWrapper" name="asyncFile">
      <target xsi:type="File"
              name="logFile"
              fileName="Logs/log-${shortdate}.log"
              layout="${longdate}|${level:uppercase=true}|${logger}|${message} ${exception:format=ToString}"
              keepFileOpen="true"
              concurrentWrites="true" />
    </target>

    <!-- 写入数据库 -->
    <target xsi:type="Database" name="database"
            connectionString="${var:connectionString}">
      <commandText>
        INSERT INTO SysLog(Level, Logger, Message, Exception, CreatedTime)
        VALUES(@level, @logger, @message, @exception, @createdTime)
      </commandText>
      <parameter name="@level" layout="${level}" />
      <parameter name="@logger" layout="${logger}" />
      <parameter name="@message" layout="${message}" />
      <parameter name="@exception" layout="${exception:format=ToString}" />
      <parameter name="@createdTime" layout="${date:format=yyyy-MM-dd HH\:mm\:ss.fff}" />
    </target>
  </targets>

  <rules>
    <logger name="Microsoft.*" maxlevel="Info" final="true" />
    <logger name="Microsoft.EntityFrameworkCore.*" maxlevel="Warning" final="true" />
    <logger name="*" minlevel="Debug" writeTo="asyncFile" />
    <logger name="*" minlevel="Info" writeTo="database" />
  </rules>
</nlog>
```

### 优点
- 性能优异，异步写入极快
- 配置灵活（XML/代码/配置文件）
- 支持 100+ Target（文件、数据库、邮件、ES、Slack 等）
- 学习曲线低，上手简单

### 缺点
- Target 生态不如 Serilog 的 Sink 丰富
- 结构化日志的表达能力不如 Serilog 的消息模板

### 适用场景
- 注重日志写入性能的项目
- 需要灵活配置日志输出目标

## 方案七：EF Core SaveChangesInterceptor（内置审计，零依赖）

### 原理
EF Core 内置的 `SaveChangesInterceptor`，无需第三方库即可实现字段级审计。比 Audit.NET 更轻量，与现有架构完全兼容。

### 实现步骤

#### 1. 创建审计拦截器

```csharp
// Infrastructure/Persistence/Interceptors/AuditSaveChangesInterceptor.cs
public class AuditSaveChangesInterceptor : SaveChangesInterceptor
{
    private readonly IHttpContextAccessor _httpContextAccessor;
    private readonly ILogger<AuditSaveChangesInterceptor> _logger;

    public AuditSaveChangesInterceptor(
        IHttpContextAccessor httpContextAccessor,
        ILogger<AuditSaveChangesInterceptor> logger)
    {
        _httpContextAccessor = httpContextAccessor;
        _logger = logger;
    }

    public override InterceptionResult<int> SavingChanges(
        DbContextEventData eventData,
        InterceptionResult<int> result)
    {
        var context = eventData.Context;
        if (context == null) return base.SavingChanges(eventData, result);

        var entries = context.ChangeTracker.Entries()
            .Where(e => e.State == EntityState.Added
                     || e.State == EntityState.Modified
                     || e.State == EntityState.Deleted);

        var currentUser = _httpContextAccessor.HttpContext?.User?.Identity?.Name ?? "System";

        foreach (var entry in entries)
        {
            var auditLog = new AuditLog
            {
                TableName = entry.Entity.GetType().Name,
                OperationType = entry.State.ToString(),
                PrimaryKey = entry.Properties
                    .Where(p => p.Metadata.IsPrimaryKey())
                    .Select(p => $"{p.Metadata.Name}={p.CurrentValue}")
                    .FirstOrDefault(),
                OldValues = entry.State == EntityState.Modified
                    ? JsonSerializer.Serialize(
                        entry.Properties
                            .Where(p => p.IsModified)
                            .ToDictionary(p => p.Metadata.Name, p => p.OriginalValue))
                    : null,
                NewValues = entry.State != EntityState.Deleted
                    ? JsonSerializer.Serialize(
                        entry.Properties.ToDictionary(p => p.Metadata.Name, p => p.CurrentValue))
                    : null,
                OperatedBy = currentUser,
                OperatedTime = DateTime.Now
            };

            context.Set<AuditLog>().Add(auditLog);
        }

        return base.SavingChanges(eventData, result);
    }
}
```

#### 2. 注册拦截器

```csharp
// Program.cs 或 DbContext 注册处
builder.Services.AddDbContext<MesDbContext>(options =>
{
    options.UseSqlServer(connectionString)
           .AddInterceptors(new AuditSaveChangesInterceptor(
               new HttpContextAccessor(),
               loggerFactory.CreateLogger<AuditSaveChangesInterceptor>()));
});
```

### 优点
- **零第三方依赖**，EF Core 内置功能
- **字段级变更记录**：记录修改前/修改后每个字段的值
- 与现有 MesDbContext 完全兼容
- 可选择性标记哪些实体需要审计

### 缺点
- 只能记录通过 DbContext 的变更（直接 SQL 不记录）
- 需要自行处理日志表查询页面

### 适用场景
- 需要字段级审计追踪
- 不想引入第三方依赖
- 已使用 EF Core 的项目（完美兼容）

### 适用场景

#### 4. 自定义遥测

```csharp
public class TProductService
{
    private readonly TelemetryClient _telemetryClient;

    public Result CreatePackOnline(...)
    {
        var startTime = DateTime.UtcNow;
        var timer = Stopwatch.StartNew();

        try
        {
            // 业务代码
            _telemetryClient.TrackEvent("PackOnlineSuccess",
                new Dictionary<string, string> { { "ProductCode", productCode } });
        }
        catch (Exception ex)
        {
            _telemetryClient.TrackException(ex,
                new Dictionary<string, string> { { "ProductCode", productCode } });
        }
        finally
        {
            _telemetryClient.TrackDependency("MES", "CreatePackOnline",
                productCode, startTime, timer.Elapsed, success);
        }
    }
}
```

### 优点
- Azure 生态原生集成，零配置
- 请求追踪、依赖监控、异常捕获、性能计数器
- Azure Portal 直接查看，无需额外部署
- Power BI 集成做报表

### 缺点
- 绑定 Azure 生态
- 数据存储在 Azure（延迟和合规问题）
- 免费额度：每月 5GB 数据摄入

### 适用场景
- 已部署在 Azure 的项目
- 需要一站式 APM（请求/依赖/异常/性能）

## 方案十：.NET Aspire Dashboard（微软最新可观测性方案）

### 原理
微软 2024 年发布的 .NET Aspire，内置 OpenTelemetry Dashboard，零配置即可查看日志、链路追踪、指标三大可观测性数据。

### 实现步骤

#### 1. 安装 NuGet

```bash
dotnet add package Aspire
# 或在新建项目时选择 Aspire 模板
dotnet new aspire-starter
```

#### 2. 配置 Program.cs

```csharp
// AppHost/Program.cs
var builder = DistributedApplication.CreateBuilder(args);

builder.AddProject<Projects.Mes_WebApi>("webapi");

builder.Build().Run();

// WebApi/Program.cs 中自动集成 OpenTelemetry
builder.Services.AddOpenTelemetry()
    .WithTracing(tracing => tracing.AddAspNetCoreInstrumentation())
    .WithMetrics(metrics => metrics.AddAspNetCoreInstrumentation())
    .WithLogging();
```

#### 3. 自动获取的 Dashboard

启动后自动打开 Aspire Dashboard：
- **日志（Logs）**：实时查看所有服务日志
- **追踪（Traces）**：请求链路追踪
- **指标（Metrics）**：CPU/内存/请求量/错误率

### 优点
- 微软官方方案，与 .NET 8 深度集成
- 零配置自动集成 OpenTelemetry
- 实时 Web Dashboard，本地和生产均可用
- 同时覆盖日志、追踪、指标三大支柱

### 缺点
- 更适合新项目（需 Aspire 项目模板）
- 目前生态还在发展中
- 需要学习 Aspire 编程模型

### 适用场景
- .NET 8 新项目
- 需要快速获得完整可观测性
- 微服务架构

## 产品日志表设计（TProductLog）

MES 系统中产品的关键操作包括：

| 上线装箱 | TProductService.CreatePackOnline | 箱体上线 |
| 模组入箱 | TProductService.CreatePackInboundModule | 模组入箱 |
| 产品下线 | BussinessController.ProductOffline | 产品下线 |
| 产品释放 | BussinessController.ProductRelease | 产品释放 |
| 产品返工 | BussinessController.ProductRework | 产品返工 |
| 归档 | TProductService.Archive | 产品归档 |

### TProductLog 实体

```csharp
// Domain/Entities/TProductLog.cs
public class TProductLog : IEntity<long>, IAuditable
{
    public long Id { get; set; }
    public int ProductId { get; set; }          // 关联产品ID
    public string? ProductCode { get; set; }    // 产品码
    public string? OperationType { get; set; }  // 操作类型（Create/Copy/PackOnline/PackInbound/Offline/Release/Rework/Archive/Restore/Delete）
    public string? OperationName { get; set; }  // 操作名称（中文描述）
    public string? Description { get; set; }    // 操作详情/备注
    public string? BeforeStatus { get; set; }   // 操作前状态
    public string? AfterStatus { get; set; }    // 操作后状态
    public string? OperateData { get; set; }    // 操作附加数据（JSON，可选）
    public string? ProductionLine { get; set; } // 产线
    public string? WorkProcedure { get; set; }  // 当前工序
    public string? UserId { get; set; }         // 操作人ID
    public string? UserName { get; set; }       // 操作人姓名
    public DateTime CreatedTime { get; set; }
    public DateTime ModifiedTime { get; set; }
    public string? CreatedBy { get; set; }
    public string? ModifiedBy { get; set; }
}
```
