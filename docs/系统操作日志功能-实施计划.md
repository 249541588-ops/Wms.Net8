# 完善系统操作日志功能 — 实施计划

> **文档日期：** 2026-06-08
> **适用项目：** Wms.Core (.NET 8 + EF Core + SQL Server + Vue 3)

---

## 一、背景与问题

当前系统操作日志存在多个问题：

| # | 问题 | 说明 |
|---|------|------|
| 1 | 前端页面 broken | `operation-log.vue` 调用 `/api/v1/Log/SystemLogs`，后端只有 `/api/v1/Log/AuditLogs` |
| 2 | 日志字段缺失 | `OperationLogFilter` 写入 `AuditLogs`，缺少 URL/状态码/耗时，OperationTime 全 NULL |
| 3 | 密码明文泄露 | 登录请求参数完整记录，包含 password |
| 4 | 菜单页面缺失 | 5 个日志菜单中只有 1 个页面存在（且 broken） |
| 5 | InterfaceOps 空壳 | 表存在但 0 条记录，无写入机制，字段简陋 |

## 二、整体方案

### 数据表职责划分

| 表 | 数据库 | 用途 | 保留策略 |
|----|--------|------|---------|
| **SystemLogs** (新建) | WmsDb (主库) | HTTP 操作日志，替代 AuditLogs + Ops | 保留 30 天 |
| **InterfaceLogs** (新建) | WmsLogsDb (独立库) | WCS/MES 接口通信日志 | 保留 60 天 |
| LocationOps (保留) | WmsDb (主库) | 用户变更货位状态记录 | 不动 |
| UnitloadOps (保留) | WmsDb (主库) | 货载操作流水 | 不动 |
| BatteryOp (保留) | WmsDb (主库) | 电芯操作流水 | 不动 |
| ~~AuditLogs~~ (废弃) | — | — | 删除 |
| ~~Ops~~ (废弃) | — | — | 删除 |
| ~~InterfaceOps~~ (废弃) | — | — | 删除 |

public class InterfaceLog : IEntity<int>
{
    public int Id { get; set; }
    public DateTime CreatedTime { get; set; } = DateTime.UtcNow;

    /// <summary>请求来源：WCS / MES / 内部定时任务</summary>
    [MaxLength(50)]

    /// <summary>接口/方法名称</summary>
    [MaxLength(100)]
    public string Endpoint { get; set; } = string.Empty;

    /// <summary>请求者/设备标识</summary>
    [MaxLength(100)]
    public string? Requester { get; set; }

    /// <summary>请求位置</summary>
    [MaxLength(50)]
    public string? LocationCode { get; set; }

    /// <summary>容器编码</summary>
    [MaxLength(100)]
    public string? ContainerCode { get; set; }

    /// <summary>请求数据体 (截断 8000)</summary>
    public string RequestBody { get; set; } = string.Empty;

    /// <summary>返回处理结果 (截断 8000)</summary>
    public string ResponseBody { get; set; } = string.Empty;

    public bool Success { get; set; }
    public long DurationMs { get; set; }
    public bool IsDuplicate { get; set; }

    [MaxLength(2000)]
    public string? Comment { get; set; }

    object IEntity.Id => Id;
}

### 步骤 6：重写 OperationLogFilter ✅

**修改文件：** `src/Wms.Core.WebApi/Filters/OperationLogFilter.cs`

核心改动：
- 写入 `SystemLog` 而非 `AuditLog`
- 设置 `OperationTime = DateTime.UtcNow`
- 记录 `Url`、`StatusCode`、`DurationMs`、`Success`
- 密码脱敏：Auth 模块请求中的 password 字段替换为 `***`

```csharp
private string? SanitizeRequestBody(string? json, string? module)
{
    if (json == null) return null;
    if (module == "Auth" && json.Contains("password"))
        return Regex.Replace(json, @"""password""\s*:\s*""[^""]*""", @"""password"":""***""");
    return json;
}
```

重复请求检测（ConcurrentDictionary 内存缓存）：
```csharp
private static readonly ConcurrentDictionary<string, DateTime> _recentRequests = new();

var fingerprint = $"{endpoint}:{locationCode}:{containerCode}";
var now = DateTime.UtcNow;
if (_recentRequests.TryGetValue(fingerprint, out var lastTime)
    && (now - lastTime).TotalSeconds < 5)
{
    log.IsDuplicate = true;
}
_recentRequests[fingerprint] = now;
```

### 步骤 9：新建 LogCleanupService ✅

**新建文件：** `src/Wms.Core.WebApi/Services/LogCleanupService.cs`

```csharp
public class LogCleanupService : BackgroundService
{
    protected override async Task ExecuteAsync(CancellationToken stoppingToken)
    {
        while (!stoppingToken.IsCancellationRequested)
        {
            var now = DateTime.UtcNow;
            var nextRun = now.Date.AddDays(1).AddHours(2);
            await Task.Delay(nextRun - now, stoppingToken);

            using var scope = _scopeFactory.CreateScope();
            // SystemLogs 保留 30 天
            var mainDb = scope.ServiceProvider.GetRequiredService<WmsDbContext>();
            mainDb.Database.ExecuteSqlRaw("DELETE FROM SystemLogs WHERE OperationTime < DATEADD(DAY, -30, GETDATE())");
            // InterfaceLogs 保留 60 天
            var logDb = scope.ServiceProvider.GetRequiredService<WmsLogDbContext>();
            logDb.Database.ExecuteSqlRaw("DELETE FROM InterfaceLogs WHERE CreatedTime < DATEADD(DAY, -60, GETDATE())");
        }
    }
}
```
